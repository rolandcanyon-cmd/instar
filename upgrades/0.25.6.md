# Upgrade Guide — vNEXT

<!-- bump: patch -->

## What Changed

- **Sleep/wake drift threshold raised from 5s to 15s** — The SleepWakeDetector was too sensitive at 5s, catching normal OS scheduling jitter (9-10s drift on a 2s interval) as wake events. At 15s, real sleeps are still detected but false positives from loaded-system timer drift are eliminated.

- **Cross-outage notification rate limit** — Added a 30-minute cooldown to "Server went down" Telegram notifications. Previously, the debounce only suppressed duplicates within a single outage; when the server flapped (down→up→down during macOS Power Nap cycles), each recovery reset the flag and fired a new notification. The cooldown persists to disk via lifeline-rate-limit.json and survives process restarts.

- **Autonomous stop hook improvements** — Better signal handling for autonomous mode sessions.

## What to Tell Your User

- **Quieter overnight notifications**: "I'll no longer spam you with server-down messages every 15 minutes when your laptop is sleeping. You'll get one notification per outage, and then silence for at least 30 minutes even if the server bounces repeatedly."

## Summary of New Capabilities

| Capability | How to Use |
|-----------|-----------|
| Cross-outage notification cooldown | Automatic — 30-minute minimum gap between server-down notifications |
| Improved sleep detection | Automatic — fewer false wake detections on loaded systems |
