# Pre-build claim check (`instar dev:claim-check`)

When several of an agent's sessions (or several agents) work in parallel, two of them can unknowingly start building a fix for the same problem — that happened twice in one night on 2026-06-05, and each collision wasted a full build-review-CI cycle. This new developer command is run BEFORE starting a build: you give it the files you plan to touch (and optional keywords), and it lists every open or recently-merged PR that touches those same files plus any design spec mentioning those keywords. If something shows up, the right move is to read it and explicitly claim a different layer of the problem — division of labor instead of two people building the same thing. It's advisory (it warns, it doesn't block), read-only, and degrades gracefully when GitHub isn't reachable.
