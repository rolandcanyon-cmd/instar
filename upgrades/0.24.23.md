# v0.24.23 — Slack Connection Stability

## What Changed

**Heartbeat timeout increased to 1 hour** — The Socket Mode heartbeat was causing reconnection loops on quiet channels. At 5 minutes, it would kill healthy connections and messages arriving during the 13-second reconnect window were silently lost. Now set to 1 hour since WebSocket close events handle real disconnections instantly. The heartbeat is only a zombie-connection detector for edge cases.

**Debug logging removed** — Verbose raw WebSocket message logging and envelope dumps added during Slack development have been cleaned up.

## What to Tell Your User

The Slack connection should be rock-solid now. Previously, quiet channels could cause the bot to briefly disconnect every few minutes, which sometimes caused it to miss messages. That is fixed — the connection stays alive for hours without interruption.
