## Summary of the change

Implements the converged + approved spec `docs/specs/duplicate-build-guard.md` (6 review rounds, cross-model every round): a concurrency-first duplicate-build detector for the instar-dev workflow, born from the 2026-07-12 ACT-562-vs-#1458 incident (a full session spent building work already merged under a different tracking id). Four pieces: (1) `scripts/lib/duplicate-build-check.mjs` — a deterministic, fail-open, security-hardened check that extracts a spec's targets (exact census/symbol/file targets + a calibrated token-set-Jaccard fuzzy floor) and scans three concurrency sources (a local sibling in-flight ledger with write-first-then-scan + liveness-conjunction + terminal lifecycle; a bounded two-stage open-PR scan; a merged-commit lookback) plus a `main`-state corroborator, emitting a TOTAL verdict ladder (`likely-duplicate`/`verify`/`clear`) with cause tagging; (2) `.claude/hooks/duplicate-build-start-gate.js` — a repo-local PreToolUse hook (Write|Edit|MultiEdit) that blocks the FIRST implementation write on a `likely-duplicate`/`verify` verdict until the author records a disposition (`{decision, reason, acknowledgedEvidenceIds[]}`), with a `check-errored` fail-open auto-stub; (3) a presence-only backstop in `scripts/instar-dev-precommit.js` (refuses a trace lacking the `duplicateBuildCheck` field only when the guard is provably live; warns loudly on `check-errored`; NEVER gates on the verdict's value); (4) a `warnings[]`-only advisory in `scripts/pre-push-gate.js`. Audit trail at `logs/dup-build-check.jsonl` (metadata only). Kill switch `INSTAR_DUP_BUILD_CHECK=off`. **Live-verified on its own spec**: the check returned `likely-duplicate (cause: concurrency)` with evidence ids pointing at the literal incident commits (`4502e4e9` #1458, `88da6009` #1461) — the origin incident geometry is provably caught.

## Decision-point inventory

- **Overlap detection** (`duplicate-build-check.mjs`) — add — a pure signal-producer: deterministic reads, emits a verdict + evidence, never blocks anything itself. `invariant` per the spec's Decision-points section.
- **Build-start PreToolUse gate** (`duplicate-build-start-gate.js`) — add — blocks the first implementation write ONLY on the absence of a recorded disposition (a structural field-validator, the signal-vs-authority "hard-invariant validation" carve-out). The proceed/abandon judgment is the author's; any recorded disposition (including proceed-on-likely-duplicate) unblocks. Fail-open on every error.
- **Precommit presence backstop** (`instar-dev-precommit.js`) — modify — presence-only field validation; accepts `decision:"proceed"` on `likely-duplicate`; warns (never blocks) on the `check-errored` stub; armed only when the guard is provably live (no retro-refusal of pre-guard traces).
- **Pre-push advisory** (`pre-push-gate.js`) — modify — appends to `warnings[]` only, never `errors[]`; honors `INSTAR_PRE_PUSH_SKIP` + `INSTAR_DUP_BUILD_CHECK=off`; a lib failure is silent (advisory absence, never a blocked push).

## 1. Over-block

The one blocking surface is the PreToolUse gate, and its over-block risk is bounded by design: it blocks only when (a) a verdict of `likely-duplicate`/`verify` exists AND (b) no disposition is recorded — and recording ANY disposition (a one-command CLI call) unblocks. A false-positive verdict therefore costs the author one look + one command, never a lost capability. Legitimate follow-on work that extends existing substrate (the #1461-extends-#1458 pattern) surfaces as `verify`/`likely-duplicate` with evidence the author acknowledges and proceeds — exactly the intended friction, dogfooded on this very build (this build's own spec triggered `likely-duplicate` because it cites the incident commits; the recorded disposition explains why it is not a duplicate). WEAK-only overlaps (common file/symbol recurrence) are a silent `clear`+note by the ladder, so the frequent-benign case never blocks or nags. The precommit backstop cannot over-block historically: it arms only when the guard was provably live for the build.

## 2. Under-block

Stated, tracked residuals (all in the spec, none silent): (a) a duplicate whose spec shares no exact target AND falls below the Jaccard threshold yields `clear` — compensated by the sibling ledger + open-PR file overlap; (b) two builds on DIFFERENT machines (or different agent homes on one box) before either opens a PR are invisible — tracked cross-machine registry `<!-- tracked: ACT-595 -->`; (c) a build whose first implementation mutation is a shell command bypasses the file-edit PreToolUse gate — tracked secondary net `<!-- tracked: ACT-600 -->`, precommit backstop still catches the PR; (d) a hard check error fail-opens with the visible `check-errored` stub (loud precommit warning) rather than blocking — the deliberate safety direction.

## 3. Level-of-abstraction fit

Correct layer: a dev-lifecycle tool wired into the existing chokepoints (the repo's own PreToolUse hooks, `instar-dev-precommit.js`, `pre-push-gate.js`, the write-trace fold) rather than a new parallel surface. The WEAK file-overlap signal reuses `scripts/lib/pre-push-scope.mjs` (no re-implemented diff). It feeds the existing authority structure — the human/agent author — rather than creating a new one.

## 4. Signal vs authority compliance

**Compliant, and reviewed as such through 6 convergence rounds.** The detector is a pure signal-producer. The two enforcement moments are structural field-validators (`docs/signal-vs-authority.md` "hard-invariant validation" carve-out): the PreToolUse gate validates a disposition EXISTS, the precommit backstop validates the field EXISTS — neither ever judges the verdict's meaning or the author's reason. A hard `git push` block was explicitly rejected in convergence (a duplicate build is recoverable, so the irreversible-action carve-out does not apply). No graded/verdict output feeds back into any routing/model/prompt decision.

## 5. Interactions

- **Precommit:** the backstop runs inside the existing gate on both Tier-1 and Tier-2 pass paths, after the trace checks — it adds one field validation, shadows nothing, and is inert when the guard isn't live. Verified by the touched-suite regression run (107/107 green incl. `instar-dev-precommit-*`, `pre-push-gate`, `write-trace-tier`, `self-action-convergence`).
- **Pre-push:** the advisory dynamic-imports the lib in try/catch — absence/failure is silent; it cannot turn a green push red (warnings[] only).
- **PreToolUse hook:** run-once semantics via a marker file (`.instar/dup-build-gate.marker.json`) — after the first allow, every subsequent tool call is a single `existsSync` (~20ms measured), so no per-edit latency tax. Instar-repo-only (package.json name check) — it never fires in other projects.
- **Write-trace:** the disposition folds into the trace fail-open; pre-guard traces round-trip byte-identically (no retro-breakage).
- **No self-action loop:** the check runs at three human/build-initiated moments (build-start, commit, push) — no timer, no tick, no self-triggered retry.

## 6. External surfaces

None beyond the repo: no HTTP routes, no Telegram, no fleet propagation. The `gh pr list`/`gh pr diff` calls are read-only GitHub queries (bounded, non-interactive, timed, CI-skipped, generic-error-on-failure — stderr never echoed). Untrusted inputs (spec text, PR titles/bodies, ledger values) are type-validated, byte-capped, control-char-stripped, and never shelled (argv arrays with `--` separators throughout; the `safe-git-allow:` marker documents the argv-only discipline in lieu of the TS SafeGitExecutor, which is unavailable in hook context without a guaranteed dist build — substance preserved, deviation recorded).

## 7. Multi-machine posture (Cross-Machine Coherence)

Per the spec's converged posture: the open-PR + merged-commit sources are **`unified`** (read from `origin/main` — fetched/normalized — and GitHub, shared truth; the cache is a worktree-scratch ephemeral, not durable per-machine state). The sibling in-flight ledger is **`machine-local` BY DESIGN** (`machine-local-justification: hardware-bound-resource` — a concurrently-building sibling process/worktree physically exists on one machine's disk; the same-agent-home scope is stated honestly, and the cross-machine early window is the tracked follow-up `<!-- tracked: ACT-595 -->`). No user-facing notices, no generated URLs, no state that strands on topic transfer.

## 8. Rollback cost

**Trivial.** `INSTAR_DUP_BUILD_CHECK=off` no-ops every layer instantly (check, hook, backstop, advisory). Full revert is a plain git revert: no data migration, no agent-state repair, no fleet coordination (Migration Parity does not apply — dev-lifecycle tooling, not agent-installed files). The ledger/audit files are append-only local artifacts that can simply be deleted. Because no layer ever gates on the verdict's meaning, a rollback cannot regress any build/commit/push behavior beyond removing the new advisories.

## Second-pass review (independent)

**Concur with the review.** Verified adversarially against the real diff, the spec, and by running the tests myself — not from the author's claims.

**A — Fail-open totality: verified.** `runDuplicateBuildCheck` wraps `runInner` in a total try/catch returning `check-errored` (duplicate-build-check.mjs:943-959); the CLI exits 0 on EVERY path including its own crash handler (:1447-1453) and usage errors (:1413). Every subprocess is deadline-bounded (`remainingMs` floors at 1ms; git ≤3s / gh ≤5s / total 8s — :342-345, :624-636, :668-677). The hook parses as ESM (package.json `"type":"module"`), and `main().catch(() => process.exit(0))` (:226-229) plus explicit fail-open exits (unreadable stdin :114, lib-unimportable → auto-stub+allow :131-142, unknown verdict :222) mean a hook crash cannot block. Precommit: `enforceDuplicateBuildBackstop` is try/catch fail-open on its own bugs, while a real refusal rides `blockCommit`'s `process.exit(1)` (uncatchable — the catch can't swallow a legit refusal into a no-op either). Retro-refusal is impossible: refusal arms only on `explicitlyOn || stubExists`; env-unset + no-stub → advisory warn only. Pre-push: the whole advisory sits in try/catch, only ever `warnings.push`; the report section exits 1 only when `errors.length > 0` — integration tests prove exit-0-with-⚠️, `INSTAR_PRE_PUSH_SKIP`, and the off-switch.

**B — Never a meaning-authority: verified, with one precision note.** Precommit reads `field.verdict` solely to WARN on `check-errored`; it blocks only on field ABSENCE and returns unconditionally when the field exists (a `proceed`-on-`likely-duplicate` and even a null-decision field pass). The hook blocks only on (a) no disposition, (b) the author's OWN recorded `abandon` (hook :194-199), (c) the §3.4 schema-completeness rule for a likely-duplicate proceed (reason + ack). Precision note: the artifact's "any recorded disposition … unblocks" overstates slightly — an `abandon` disposition keeps blocking (correct per spec: it enforces the author's own decision, not the verdict's meaning; re-record proceed is one command).

**C — Hardening real: verified.** grep-confirmed zero `execSync`/shell-string subprocess calls in the new code (6 callsites, all `execFileSync`/`spawnSync` argv arrays); typed validators reject leading-`-` tokens and non-repo-relative paths BEFORE argv, with `--` separators on grep/ls-tree; byte caps enforced at ingest (:660) AND re-enforced at the similarity site (:1107, covers injected/fixture sources); gh stderr is piped but never read or echoed (generic notes only); `git grep` pathspec-scoped to the census file + `src/` with a 50-match cap + 300-char clamp; sibling `worktreePath` gets lstat-symlink-reject + realpath jail under allowed roots (:443-472); `clampUntrusted` strips control/escape chars on every evidence string.
