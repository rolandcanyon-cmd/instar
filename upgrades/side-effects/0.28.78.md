# Side-Effects Review — v0.28.78 (Token Ledger bounded scan fix)

**Version:** `0.28.78`

**Author:** Roland
**Second-pass reviewer:** not required (performance fix only; no decision-point surface)

## Summary of the change

Bounds the Token Ledger's initial scan via a per-tick file cap (500 files/tick), intra-tick yielding via `setImmediate` every 25 files, and an optional max-file-age cutoff (30 days). Fixes an event-loop block on agents with large JSONL transcript histories that caused the lifeline supervisor to restart-loop the agent.

Files touched:
- `src/monitoring/TokenLedger.ts` — bounded scan, cursor persistence, per-tick yielding
- `src/monitoring/TokenLedgerPoller.ts` — wires new scan options

## Decision-point inventory

No decision-point surface. The fix changes *when* files are scanned (batched/yielding), not *what* is done with the data. No block/allow logic, no message filtering, no session-lifecycle mutation.

## 3. Level-of-abstraction fit

Pure performance fix to an existing monitoring component. The bounded-scan approach (cursor + per-tick cap + age cutoff) is a standard backfill pattern. No new abstractions introduced.

No change to signal/authority surface. The ledger remains pure read-side observability.

## 5. Interactions

- **Event loop:** Yields every 25 files, keeping the server responsive during scan.
- **Scan completeness:** The cursor persists across ticks so all files are eventually scanned — just spread across multiple 60s intervals rather than one blocking pass.
- **Age cutoff:** Files older than 30 days are skipped during backfill. Active sessions (any new appended turn updates mtime) are always included. The source JSONLs remain the ground truth.
- **No new races, feedback loops, or cross-agent interactions.**

## 6. External surfaces

None. Pure internal performance fix.

## 7. Rollback cost

Revert the commit. The ledger will resume unbounded scanning (the original behavior), which is safe for agents without large transcript histories. No persistent state changes.

Pure performance fix with no decision-point surface, no external interactions, and no behavioral changes. Clear to ship.
