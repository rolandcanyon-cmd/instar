# Side-Effects Review — v1.3.619 fork CI fix

**Scope:** Fork-only CI change. Not an instar runtime change.
**Files:** .github/workflows/publish.yml

## What changed

One-line fallback added to the fork's `publish.yml`:
- `token: ${{ secrets.RELEASE_TOKEN }}` → `token: ${{ secrets.RELEASE_TOKEN || github.token }}`

Keeps fork CI green when `RELEASE_TOKEN` secret is absent. The `github.token` fallback will fail if branch protection is enabled, but allows the publish workflow to proceed in environments without the PAT configured.

## Blast radius

- **CI-only.** No changes to `src/`, `dist/`, hooks, skills, jobs, or any runtime surface.
- **No agent behavior change.** End users and agents are unaffected.
- **Fork-specific.** Upstream JKHeadley/instar is not changed.

## Risk + mitigation

- **Risk:** `github.token` may lack permissions to push version bump commits if branch protection is on. **Mitigation:** this is the pre-existing upstream behavior without the secret — the fallback doesn't make things worse, it makes explicit what was implicit.

## Rollback

Remove the `|| github.token` fallback from `publish.yml`. No agent restart or migration needed.
