# v0.26.2 — Fix Unsolicited Session Health Messages

## What Changed

**SessionMonitor only acts when user is waiting** — Previously, the SessionMonitor would trigger health checks, nudges, escalation, and session restarts on idle sessions even when the agent had already responded and was just waiting for the next user message. This caused noisy unsolicited messages in Slack channels about session health. Now, the monitor only escalates when the user has sent a message that has not been answered.

## What to Tell Your User

Fixed an issue where the bot would post unsolicited messages about session health and recovery even when you were not waiting for a response. The bot will now only check on sessions when you have an unanswered message.

## Summary of New Capabilities

| Capability | How to Use |
|-----------|-----------|
| Quiet idle sessions | Automatic — no action needed |
