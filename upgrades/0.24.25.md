# v0.24.25 — Fix Slack Standby Spam

## What Changed

**Removed Telegram-to-Slack standby mirroring** — Previously, Telegram standby messages were mirrored to Slack if the session had a Slack channel. This caused standby updates to appear in Slack even when no one had sent a message there. Now Slack only gets standby from its own direct PresenceProxy wiring, which correctly triggers only after a user message in that Slack channel.

## What to Tell Your User

Fixed an issue where status updates were appearing in Slack channels even when you had not sent a message there. Standby updates now only appear in the channel where you actually sent a message.

## Summary of New Capabilities

| Capability | How to Use |
|-----------|-----------|
| Platform-isolated standby | Automatic — each platform only shows standby for its own conversations |
