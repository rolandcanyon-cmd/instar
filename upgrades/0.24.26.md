# v0.24.26 — Fix Slack Standby Cancellation

## What Changed

**Standby cancellation on agent response** — The PresenceProxy never saw agent responses in Slack because bot messages returned through Socket Mode with bot_id set, getting filtered before reaching the event handler. This meant standby timers ran indefinitely even after the bot had responded. Now the reply route notifies the PresenceProxy when the agent sends a response, properly cancelling the standby cycle.

## What to Tell Your User

Fixed an issue where the status updates kept appearing in Slack even after the bot had already responded. The status updates now properly stop once the bot sends its reply.

## Summary of New Capabilities

| Capability | How to Use |
|-----------|-----------|
| Proper standby cancellation in Slack | Automatic — standby stops as soon as the agent responds |
