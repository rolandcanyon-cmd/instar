# Upgrade Guide — v0.12.5

<!-- bump: patch -->

## What Changed

Silent updates: auto-updates are now completely invisible to the user unless active sessions will be interrupted.

Three notification sources silenced:

1. **AutoUpdater.gatedRestart()** — when no sessions are running, restart is now silent (console.log only, no Telegram). Previously always sent "Just updated to vX. Restarting to pick up the changes." even when nobody was working.

2. **AutoUpdater.applyPendingUpdate()** — when no restart is needed, the "Just updated to vX" notification is now silent. Previously always sent a Telegram message.

3. **Post-restart upgrade guide delivery** — the dedicated Claude session (UpgradeNotifyManager) that spawned after restart to message the user about "what's new" is disabled. The pending upgrade guide file is preserved and injected into the agent's context at the next natural session start via ContextHierarchy. No separate notification session is spawned.

Notifications that STILL fire (correctly):
- Pre-restart warning when sessions ARE active ("Server restarting in N seconds, M sessions will resume")
- Forced restart after max deferral when sessions won't finish
- Deferral warnings (first warning + final warning) for active sessions
- Update failure notifications (so the user knows something went wrong)

## What to Tell Your User

- **Silent updates**: "Updates now happen completely in the background. You'll only be notified if an update needs to restart the server while you have active sessions running."
