# Upgrade Guide — v0.24.18

## What Changed

### Systems Dashboard v3 — Rich Card Grid with Detail Views

The Systems tab has been completely redesigned from a cramped pill grid to an interactive card-based layout with drill-down detail views.

**Overview page:**
- Responsive card grid (2 columns on desktop, 1 on mobile)
- Each card shows the capability name, a 2-line description, key stat chips, and a live metric summary
- Issues panel only appears when there are actual problems
- Events section collapsed by default to reduce noise

**Detail views:**
- Click any capability card to see its full detail page
- Stat cards grid showing all numeric metrics individually
- Component process list with status indicators
- Last activity timestamp
- Back navigation to return to overview

**Rich telemetry backend:**
- Each capability now returns live metrics: recoveries, interventions, coherence checks, memory pressure, quota usage, topic counts, evolution stats
- New `GET /systems/capability/:id` endpoint for individual capability details
- Richer capability descriptions (2-3 sentences each)

### SlackAdapter: Missing Methods Fixed

The SlackAdapter was missing several methods (channel-session registry, resume map, stall detection callback) that server routes depended on. This caused TypeScript build failures in CI. The methods are now committed and the build is clean.

## What to Tell Your User

The Systems dashboard got a major upgrade. Instead of a flat grid of status pills, it now shows interactive cards with live metrics for each capability — things like how many session recoveries happened, current quota usage, and Telegram connection uptime. You can click any card to drill into the full details. It's a much clearer picture of what your agent is actually doing at a glance.

## Summary of New Capabilities

- **Systems dashboard v3:** Interactive card grid with drill-down detail views and live metric summaries
- **Per-capability telemetry:** Each system reports real-time stats (recoveries, quota %, topic counts, etc.)
- **Capability detail endpoint:** `GET /systems/capability/:id` for individual system deep-dives
- **SlackAdapter completeness:** All adapter methods now committed, CI build clean
