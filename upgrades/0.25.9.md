# v0.25.9 — PromptGate Enabled by Default

## What Changed

**PromptGate enabled by default for all agents** — Previously, PromptGate (prompt detection, auto-approval, and user relay) was not included in the default agent config. This meant permission prompts in sessions were invisible to users, causing sessions to hang indefinitely. Now enabled automatically for both new agents (via init) and existing agents (via post-update migration).

**Post-update migration** — Existing agents that update to this version will automatically get PromptGate enabled with sensible defaults: file creation and file edits are auto-approved, plan approvals are relayed to the user for decision.

## What to Tell Your User

Permission prompts that appear in your agent sessions are now automatically handled. File edits and creations are approved automatically. Plan mode prompts are sent to you for approval via your messaging platform. This prevents sessions from getting stuck waiting for input that you could not see.

## Summary of New Capabilities

| Capability | How to Use |
|-----------|-----------|
| Auto-approve file edits | Automatic — no action needed |
| Plan approval relay | Automatic — prompts appear in Telegram/Slack |
| Migration for existing agents | Automatic on update |
