# Upgrade Guide — vNEXT

## What Changed

### Git Sync Job — Now Enabled by Default

The `git-sync` job is now **enabled by default** for all agents. Non-technical users don't need to know about it — it just works in the background.

**How it works invisibly:**
- Gate script silently skips if there's no git repo or no remote configured
- Gate skips if nothing needs syncing (no local or remote changes)
- Only spawns a Claude session when actual sync work is needed
- Zero cost when idle — no session, no tokens

**What changed from v0.9.32:**
- `enabled: true` by default (was `false`)
- Gate now checks for remote existence — agents without a remote never trigger a sync
- Existing agents: `refreshJobs` auto-enables git-sync if git + remote are detected

**For existing agents upgrading:** If you had git-sync disabled from the previous version and your project has a git remote, it will be auto-enabled on next update. If you have no git remote, nothing changes.

## Summary of New Capabilities

- **git-sync auto-enabled**: Works invisibly for agents with git + remote
- **Remote detection in gate**: Prevents wasteful sessions for repo-only (no remote) agents
- **Auto-enable migration**: Existing disabled git-sync jobs get enabled when git + remote detected
