# v0.24.29 — Fix Stuck Slack Sessions

## What Changed

**Kill stuck sessions instead of losing messages** — When a Slack session gets stuck on a permissions prompt or tool hang, the server previously injected the user's message anyway, which silently failed. Now it detects the stuck session, kills it, and spawns a fresh one so the user's message is always processed. This prevents the scenario where a user sends messages and gets checkmarks but never receives a response.

## What to Tell Your User

Fixed an issue where messages in Slack could get lost if the bot session was stuck. The bot now automatically recovers by restarting the session when it detects a problem, so your messages always get processed.

## Summary of New Capabilities

| Capability | How to Use |
|-----------|-----------|
| Automatic stuck session recovery | Automatic — stuck sessions are killed and respawned on new messages |
