# Convergence Check Own Tunnel URL Allowlist — Plain-English Overview

> The one-line version: links created by the agent on its own tunnel should pass the pre-message URL check, while made-up outside domains should still be stopped.

## The problem in one breath

Before an agent sends a message, a small script checks for common mistakes. One of those checks catches unfamiliar URLs, because agents sometimes invent plausible-looking links that were never produced by a tool. That check is useful, but it was too strict for links the agent itself had just created, such as a Secret Drop form or a private-view page on the agent's tunnel.

## What already exists

- **The convergence check** — a fast shell script that runs before external messages and flags risky patterns like fabricated URLs.
- **Secret Drop** — creates a one-time link where a user can safely provide a credential.
- **Private views** — create local or tunneled pages the user can open in a browser.
- **Tunnels** — expose the agent's local server through a configured hostname or a Cloudflare quick-tunnel hostname.

## What this adds

This adds a narrow runtime exception to the URL provenance check. The script reads the agent's own tunnel hostname from its config file and treats URLs on that exact host as familiar. It also treats Cloudflare quick-tunnel hosts as familiar, because those are generated by the tunnel provider and change from run to run.

The change does not bless every tunnel-looking domain. A random domain is still flagged unless it is the exact configured hostname, localhost, loopback, a Cloudflare quick-tunnel host, or an already-known public service domain.

## The new pieces

- **Configured-host read** — the script reads the tunnel hostname from the agent's config at runtime. If that read fails, the script keeps working with the old static allowlist.
- **Quick-tunnel host match** — hosts ending in the Cloudflare quick-tunnel domain pass, because those are expected to be ephemeral and generated.
- **Regression tests** — tests cover the configured host, quick-tunnel host, and missing-config fallback while keeping fabricated domains blocked.

## The safeguards

**Keeps fabricated domains blocked.** The allowlist is still host-based and narrow. Made-up domains remain unfamiliar.

**Fails closed on config trouble.** If the config file is missing or unreadable, the script does not crash and does not broaden the allowlist.

**Uses the shipped template source.** The fix is made in the script template that PostUpdateMigrator installs, so existing agents receive the corrected script during migration instead of needing a separate manual edit.

## What ships when

This ships as one small infrastructure patch with tests and release notes. Once installed, agents can send their own Secret Drop and private-view tunnel links without tripping URL provenance, while unrelated unfamiliar links continue to be reviewed.

## What you actually need to decide

Approve the narrow URL-provenance allowlist so self-generated tunnel links can be delivered without weakening protection against fabricated external URLs.
