# Spec — ESM-Safe Instar Hook Generators + Structural Prevention

## Problem

`instar` ships ~17 command hooks generated by `PostUpdateMigrator`'s `get*Hook()` methods. Eight of them currently emit `.js` content that uses bare top-level `require(...)` (either directly or via the `const _r = require; const fs = _r('fs')` indirection). instar agent homes default to `"type": "module"` (the canonical checkout itself, and effectively every agent installed via `npx instar`). In an ESM-mode host, a `.js` file is ES module — bare top-level `require` throws `ReferenceError: require is not defined in ES module scope`. The hook **crashes on every fire**. The crash exits non-zero but Claude Code treats it as non-blocking, so the failure is **invisible**. The guardrails look installed; they're silently dead.

**Concrete impact, confirmed in production on 2026-05-27** (silent-stall incident, see `memory/bug_instar_hooks_esm_require_crash`):
- `stop-gate-router.js` (Stop hook — the **UnjustifiedStopGate router**, specifically designed to catch unjustified stops and either continue the session or record why) → crashed every Stop → no protection against silent stalls.
- `slopcheck-guard.js` (PreToolUse) → crashed on every tool call → no slop-package nudges.
- `post-action-reflection.js` (PostToolUse) → crashed on its trigger → no learning capture.
- Five additional generators (`getScopeCoherenceCollectorHook`, `getScopeCoherenceCheckpointHook`, `getClaimInterceptHook`, `getResponseReviewHook`, `getClaimInterceptResponseHook`) emit the same broken pattern.

This is a fleet-wide guardrail outage: every ESM-mode agent (which is most agents, including the canonical checkout) had eight guardrails silently failing.

## Root Cause

The codebase already had the correct fix pattern documented in `getHookEventReporterScript` — its header comment explicitly explains:
> Uses `await import('node:http')` instead of `require('http')` so this script works regardless of the host package.json's module type. A plain `require` throws in ESM scope (when the host has "type": "module"); a plain `import` is a syntax error in CJS scope. Dynamic import works in both.

Eight other generators landed with bare `require` and **nothing structurally caught the regression**. No test asserted that generated hook content is ESM-safe.

## Solution

### Two parts, both required:

**(1) Source fix.** Rewrite each of the eight broken `get*Hook()` generators to emit ESM-safe content:
- Wrap the hook body in an async IIFE: `(async () => { ...body... })();`
- Replace top-level `const _r = require; const X = _r('Y');` (and bare `const X = require('Y');`) with dynamic `const X = await import('node:Y');` inside the IIFE.
- Convert `process.stdin.on('data', ...).on('end', ...)` stdin handlers to the for-await-of pattern: `for await (const chunk of process.stdin) data += chunk;` — works in both CJS and ESM hosts.
- Add the ESM-SAFE header comment referencing `hook-event-reporter.js` for future contributors.

**(2) Structural prevention.** Add a regression test at `tests/unit/no-bare-require-in-generated-hooks.test.ts` that:
- Source-scans `PostUpdateMigrator.ts`, extracts each `get*Hook()` method body, finds backtick-template literals, and asserts none contains the forbidden bare top-level `require` patterns.
- Emits **one test per generator** so the failure surfaces exactly which generator needs the rewrite (not one opaque "8 things fail").
- Lives in `tests/unit/` (gate-exempt from the instar-dev gate; ships freely as immediate prevention).

**The regression test is the keystone**: without it, the bug class can re-ship the next time a contributor copy-pastes the wrong pattern. With it, CI fails before merge.

## Test Plan

- Tier 1 (unit): the regression test itself. Demonstrably starts at 8 failures (one per broken generator), goes to 18 passing after the source fix.
- Tier 2 (integration): manual runtime smoke — pipe a representative JSON payload through each generated `.js` content in an ESM-mode test agent home and assert no `ReferenceError`. (Performed against the installed copies during diagnosis: stop-gate-router, slopcheck-guard, and post-action-reflection all exited clean after the same fix pattern was applied.)
- Tier 3 (lifecycle): the `migrateHooks` path always-overwrites instar/ hooks (existing standard, `CLAUDE.md` Migration Parity), so the fix ships to every agent on next update automatically — no separate migration step needed.

## Migration Parity

**No new migration code required.** The fix is to the source content of generators called from `migrateHooks`. The existing always-overwrite policy for `instar/` hooks (documented in `CLAUDE.md` under Migration Parity Standard, specifically as the fix pattern for the prior `hook-event-reporter.js` ESM bug) automatically ships the corrected hook content to every agent on its next update run. Existing broken installed copies are overwritten with the ESM-safe versions.

## Side Effects

- **Hook scripts now use async IIFEs** instead of top-level scripts. Functionally equivalent for these short scripts; the async wrapper has no runtime cost beyond a single micro-task tick on startup.
- **No public API changes.** Hook contracts (stdin JSON input, exit code semantics, stdout JSON output) are preserved.
- **No config changes.** Migration is fully automatic; no settings updates required.

## Rollback

Trivial: revert the commit. The regression test would then fail; revert it too. Worst case: agents go back to the silently-broken state, which is what they had before this fix.

## Tracked Refinements (NOT in this spec)

- The other postmortem findings (notify-on-stop, false-blocker interceptor, Agent Self-Propagation harness — CMT-560) are each their own spec/PR (next tasks in the autonomous run).
- Future hooks landing as new `get*Hook()` methods are automatically covered by the regression test (one test per generator).

## Why this should ship fast

This is a clear crash bugfix with a documented fix pattern already in the codebase + a regression test that prevents recurrence. No design ambiguity. The instar-dev gate requires user approval for any `src/` commit; this spec is the request for that approval.
