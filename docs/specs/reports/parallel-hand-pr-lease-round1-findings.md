# Parallel-Hand PR Lease — Round 1 findings (consolidated)

Round 1 reviewers: 6 internal (security, scalability, adversarial, integration/multi-machine, decision-completeness, lessons-aware) + Standards-Conformance Gate (0 flags) + external codex-cli/gpt-5.5 (verdict: SERIOUS ISSUES) + external gemini-cli/2.5-pro (pending).

## Blockers (must resolve normatively in §3 before build)

- **B1 (lessons-aware, CRITICAL) — Enforcement chokepoint is WRONG.** The #1183 collision was competing commit *force-pushes*. The spec gates `safe-merge.mjs` (which only ARMS auto-merge, never pushes commits) + SKILL Phase-7 prose (Willpower; the SKILL.md doesn't even exist in this worktree). The real chokepoint is `src/core/SafeGitExecutor.ts` — the single-funnel for all destructive git ops incl. `push` (allowlist line 72). Move the lease gate THERE, in code. Verify every hand commit-push routes through SafeGitExecutor; any raw `git push` bypass is the gap to close first. Prose = awareness only, never enforcement.
- **B2 (adversarial + lessons + decision-completeness) — Hand identity = topic-id, not session-id.** instar respawns sessions constantly (CompactionSentinel, context-wedge, /sessions/refresh, resume-queue revival), each keeping the same topic but getting a new tmux name/uuid. Keying ownership on session-id → a respawned hand reads its OWN lease as foreign-live and yields to itself forever (the 30m wedge, on every rework respawn). FIX: `holderTopicId` = the stable logical owner; `holderSessionId` used ONLY as a liveness-probe handle. A hand whose topic matches the lease ALWAYS treats it as "mine" (renew+proceed, never yield).
- **B3 (adversarial) — Double-drive on takeover.** O_EXCL protects first-CREATE on a clean slate, NOT contended steal of an existing lock. Auto-heal-by-delete-then-recreate lets two hands both succeed in sequence (double-drive — the original bug). FIX: atomic CAS on the stale record (compare holder+acquiredAt, swap to self) or `rename()`-based steal; the O_EXCL loser re-reads and yields. Inherit ResumeQueue FD4 (`'wx'`, EEXIST→re-evaluate, never blind-overwrite) verbatim.
- **B4 (security + adversarial + codex#2) — Fail-closed-to-yield contradicts "can never wedge".** If the liveness probe ITSELF fails (tmux errors, SessionManager not yet repopulated post-restart), every hand fails-closed → yields → no hand ever auto-heals → guaranteed full-TTL wedge with no live driver. Distinguish "uncertain about the holder" (yield) from "liveness probe unavailable" (degrade to TTL-only takeover, made safe by B3). Add an absolute lease-age ceiling so a lease past a hard wall heals even when liveness is unknowable. State the real worst-case bounded delay explicitly.

## Material

- **M1 (security + adversarial + lessons) — Unbounded renewal by a live-but-stalled holder → permanent yield.** Auto-heal only covers a DEAD holder; a live-but-wedged holder renews forever and the second hand yields forever. Add absolute max-hold ceiling (~90m) independent of renewal → forcibly releasable + attention item (mirror PromiseBeacon boot-cap).
- **M2 (security) — Route auth.** `GET /pr-leases` must be Bearer-auth-gated like every route (exposes session/topic/branch/machine topology). Integration test asserts 401 without token.
- **M3 (scalability) — Collapse to ONE process-wide lock** on `pr-hand-leases.json` (matching ResumeQueue's single `lockPath`), drop per-PR `.lock` sidecars (unbounded dir growth; ResumeQueue holds ONE lock, not per-entry). Contention domain is the single JSON file.
- **M4 (scalability) — TTL-gate the liveness probe.** Only probe when `now - renewedAt > ttlMs`; read SessionManager's in-memory running set, NOT a `tmux` exec. Common path = single JSON read, no probe, no process spawn.
- **M5 (scalability) — Renewal interval + concurrent count.** State renewal interval (TTL/3 = 10m for 30m TTL); concurrent leases bounded by `autonomousSessions.maxConcurrent` (default 5); single-file CAS acceptable precisely at this scale.
- **M6 (integration + lessons, MULTI-MACHINE) — `holderMachineId` must be load-bearing.** The collision unit is the shared GitHub PR — inherently cross-machine. Machine-local v1 does NOTHING across machines, AND §3.5 dead-holder check MISFIRES across machines (holder on machine B is trivially absent from machine A's session set → false "dead" → double-drive). FIX: a holder on a DIFFERENT `holderMachineId` is NEVER judged dead from local session absence (fail-closed → treat live → yield). State the residual cross-machine risk honestly (§2/§8). Cross-machine replicated lease (inheriting FD1/FD2/FD5) tracked §9.
- **M7 (adversarial) — `intent` gameable.** Closed enum {build,rework,merge}, default `build`. MergeRunner soft-hold ONLY on a *live* `intent:rework`, bounded by lease TTL (a quiet rework hand can't stall auto-merge forever; a missing/expired lease NEVER blocks the watcher). Audit intent transitions.
- **M8 (adversarial + scalability) — Tombstone semantics.** Terminal status {released,merged,abandoned}; explicit ≤5s window; a successor acquires THROUGH a tombstone without yielding (a tombstone is NEVER a live holder). Prune opportunistically on each CAS write (zero extra I/O).
- **M9 (decision-completeness) — Resolve all 6 Open questions into Frontloaded Decisions** (a spec cannot converge with live user-decisions parked). Q1→ see codex#3 below; Q2→hard-refuse at the push point BUT reconcile with Signal-vs-Authority (self-coordination carve-out; the refuse must surface holder+intent so a genuinely-distinct second-hand fix is escalated as a follow-up, never silently DROPPED); Q3→machine-local v1 (decided non-goal); Q4→MergeRunner soft-hold on same-machine live rework; Q5→session-liveness AND TTL, probe-unavailable→TTL-only; Q6→topic-id identity.
- **M10 (scalability) — Fail-OPEN on unreadable state file** (distinct from fail-closed on liveness). Corrupt/missing `pr-hand-leases.json` → treat as no lease, push proceeds, log. Matches ResumeQueue `load()` reset-on-parse-fail. A missing lease file must NEVER block all pushes.
- **M11 (lessons + adversarial) — TTL vs a live-but-slow rework (>30m rebuild+CI).** The AND-liveness rule covers it (a live session keeps its lease past TTL). Specify the heartbeat source for a skill-driven (non-daemon) push sequence — if none, the liveness probe (session present) IS the keepalive; TTL is just the dead-holder backstop.
- **M12 (multiple + codex#4) — Worktree-collision half is deferred; the spec OVERCLAIMS.** §1.1 lists shared-worktree edit-bleed as a co-equal cause; the per-PR push lease does NOT solve it. Reword the header/ELI16 ("last structural cause") to "the push-competition cause"; state the residual shared-worktree risk in §2 non-goals; name the `instar worktree create` per-agent-isolation convention as the structural mitigation for the worktree half.
- **M13 (decision + codex#5) — dryRun reproduces the thrash.** `dryRun:true` logs the would-yield but still pushes → ZERO collision protection (both hands push) → the soak window can reproduce #1183 on the dev agent. The LIVE hand must still WRITE its lease (acquisition NOT dryRun-gated; only the yield-DECISION is) so a peer can see it. Bounded soak + a circuit-breaker attention item if N would-yields observed. (codex alt: dry-run could still refuse the push when a foreign LIVE lease is detected — a "shadow that still protects" — consider.)
