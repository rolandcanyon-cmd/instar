## PREMISE CORRECTION (lessons-aware BLOCKER — do this first)
The spec's problem statement ("watcher keeps a foreground child alive ~18–25 min waiting for slow CI, gets killed") is **wrong for the MergeRunner**. `classifyCandidate` (`greenPrLogic.ts:70`) only marks a PR eligible when `statusRollup === 'SUCCESS'` — the watcher NEVER acts on a pending PR, so `safe-merge`'s poll loop exits on its first iteration. The "~18-min watcher kill" in the run notes was the *manual background `safe-merge` watchers* (background Bash with a deadline), NOT the in-server MergeRunner. **Rewrite the problem statement honestly:** the real value of arm-and-handoff is (a) freeing the single-flight/lease slot in seconds instead of holding a ~25-min child, and (b) surviving a server-restart-mid-merge (the watcher's own merge triggers a release→restart that kills the in-flight `--admin` child; native auto-merge means GitHub completes it). This is a genuine but more modest benefit. Per `[[challenge-the-mechanism]]`: verify the mechanism is real before converging.

## BLOCKERS
1. **Head-pin does NOT bind through native auto-merge** (security #1, adversarial M3). `--match-head-commit` pins only at ARM time; GitHub native auto-merge merges head-at-green, and per GitHub docs only auto-disarms on pushes by users WITHOUT write permission. A write-capable push (the agent's own automation, a maintainer) after arming merges an unverified/protected-path head. The spec's "stricter than `--admin`" claim is true ONLY for required-check enforcement, FALSE for the head-pin dimension. **Fix:** either (a) empirically verify whether `enablePullRequestAutoMerge` persists + re-enforces `expectedHeadOid` / cancels on push, OR (b) honestly document the residual race + add reconciliation `mergeCommitOid` vs `armedHead` mismatch detection (it already stores `armedHead` but §74 reaps MERGED without comparing).
2. **Re-arm thrash — armed PRs not excluded from gather()** (adversarial B1, scalability #1/#2/#5). An already-armed-but-green PR re-enters `classifyCandidate` (eligible:true, since statusRollup SUCCESS) and is re-armed every tick, resetting the 24h clock + burning spawns. **Fix:** add an explicit "skip: already-armed" candidate verdict keyed on `episodes[pr].armedAt` set; an armed PR is GitHub's until reconciliation clears it.
3. **Disarm/rollback can't recall an in-flight armed merge** (adversarial B2, security #3, lessons #5). `POST /green-pr-automerge/rollback` + emergency-pause stop NEW arming but do NOT un-arm; the spec's HOLD-label workaround does NOT work (GitHub auto-merge gates on required checks, not title/labels). The operator's kill switch silently fails for the most dangerous case. **Fix:** rollback/emergency-pause AND an explicit per-PR operator HOLD must call `gh pr merge <pr> --disable-auto` on every `armedAt` episode (operator-authorized, audited — not a "surprising mutation"), + raise one attention line listing disarmed PRs.
4. **Multi-machine: armed episodes are machine-local, strand on a lease move** (integration B1). `green-pr-automerge.json` is machine-local-by-design and does NOT replicate; a lease move means the new holder never sees the armed episode and re-arms. **Fix:** make GitHub-side `autoMergeRequest` the source of truth for "already armed" — add `autoMergeRequest` to the `refetchPr`/`prState` projection; gather() skips any PR with auto-merge armed on GitHub regardless of local episode state.
5. **24h ceiling silently drops a still-armed PR** (lessons #4, scalability #2, adversarial #7). Clearing `armedAt` at the ceiling forgets a real in-flight auto-merge (under-counts merges; Close-the-Loop violation — the watcher's view closes but the loop doesn't). **Fix:** transition to `armed-overdue` that keeps reconciling + re-surfaces on a deduped cadence; close only on MERGED/CLOSED or operator action.

## KEY MATERIALS
- Reconciliation must read `mergeCommitOid` + compare to `armedHead` (security #2, lessons #2).
- `UNKNOWN`/read-failure fail-open MUST be enumerated: leave episode armed, NO ladder advance, NO breaker feed, retry next tick (scalability #3/#4, adversarial M6, lessons #3b).
- `prState` seam lacks `autoMergeRequest` — can't detect disarm; widen the projection (integration M1).
- Pin exact `armed` episode field-state: `state:'active'`, `attempts` UNCHANGED, `nextEligibleAt` CLEARED, `armedAt`/`armedHead` set (adversarial M5).
- Map `error:auto-arm-unconfirmed`/`error:auto-confirm-unreadable` to a non-ladder retry (do the reconciliation read), not a merge-failure that exhausts `maxAttempts` while GitHub is silently armed (adversarial M4).
- `mergeTimeoutMs` invariant is vestigial on the auto path → a hung arm spawn = ~26-min breaker blast radius; add `armTimeoutMs` (~60s) for the auto path; scope the 25-min invariant to `mergeStrategy:'admin'` (adversarial #8, integration m3).
- Add `mergeStrategy:'auto'` + `armedConfirmCeilingMs:86_400_000` to the `DEFAULTS` block; CLAUDE.md template awareness (integration M2; Agent Awareness + Migration Parity).
- `auto-merge-unavailable` (repo setting off) is a PERMANENT condition → classify terminal-non-ladder (one attention line), not ladder-exhaustion (integration m4).
- Promote `armedCount`/`armed:[]` to non-optional in `GET /green-pr-automerge` (integration M3, lessons #8 Observable Intelligence).
- Document the code-level rollback runbook: old code redundantly `--admin`-merges already-armed PRs — disarm them first OR document the benign race (integration B2).
