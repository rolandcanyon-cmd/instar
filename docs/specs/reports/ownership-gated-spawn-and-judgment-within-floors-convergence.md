## ELI10 Overview

On July 10 one of the operator's machines crashed for half an hour. Messages for its conversations arrived at the other machine, whose routing brain correctly said "not mine — queue it." Six milliseconds later, older code ignored that ruling and started local copies anyway. When the crashed machine came back, there were two live copies of each conversation, and every safety rule that normally prevents killing a session — each one correct on its own — combined to make the duplicates unkillable. A human had to clean it up. It was the second time in a week.

This spec fixes that three ways. First, prevention: one checkpoint that every session-creating code path must pass, which makes the routing verdict binding — a machine that doesn't own a conversation can never start a bootleg copy of it; if the owner is temporarily dark, the user gets held-then-honest-notice behavior with a hard 10-minute silence ceiling, never silence and never a lie about whether the message was saved. Second, healing: a reconciler on the lead machine that spots duplicates, determines the rightful owner from real evidence (never "who got the last message" — during this bug the wrong copy gets the last message), repairs the ownership records everywhere, verifies the other machine actually saw the repair, and lets the existing safe cleanup close the spare — after re-checking at closing time that the survivor is genuinely alive, and after handing over any open promises so nothing the agent committed to gets silently dropped. Third, a new pattern: the two genuinely fuzzy judgment calls in this flow (how long to wait on a wobbly machine; which duplicate survives on messy evidence) can be delegated to an LLM arbiter — but only inside deterministic rails, with a conservative default, mechanical corroboration for anything irreversible, full decision logging, outcome grading against reality, and a rule that the arbiter only gets authority after shadow data proves it beats the simple static rule.

The main tradeoffs, stated honestly: the design uses instar's existing coordination machinery rather than adding an external consensus system, which means ownership is NOT perfectly guaranteed during network partitions or record-store errors — divergence in those windows is bounded, loud, and self-converging rather than impossible. And when ownership records are unreadable, the system deliberately fails toward answering the user (a bounded, breaker-limited duplicate is better than silence).

## Original vs Converged

**Originally**, the auto-heal converged the ownership record and assumed the existing cleanup would then close the spare copy. Review proved that assumption wrong against the incident itself: the cleanup re-checks every keep-guard fresh, so the spare's open commitment would STILL have made it immortal — the exact July-10 outcome. **Converged:** a narrow, provenance-fenced carve-out for exactly the two topic-bookkeeping guards, valid only with a terminate-time proof that the surviving copy is live, plus an acknowledged cross-machine handoff of open commitments (redesigned twice: the first "re-bind" referenced a session field that doesn't exist on commitment records; the final design mints a successor on the owner machine and terminally supersedes the origin record only on acknowledgment, behind its own independently-reversible rollout flag — until that flag flips, commitment-shielded duplicates escalate to the operator instead of auto-closing).

**Originally**, the reconciler trusted the replicated journal to carry its repairs to the other machine. Reviewers pointed out that journal is the very channel whose bloat caused the ancestor incident. **Converged:** the reconciler confirms each peer actually OBSERVED the repair within a bounded window, escalates when it didn't, and never acts on cached/stale rows — every record write requires a fresh direct probe.

**Originally**, "most recent user interaction" was an ownership tie-breaker. Both external models and the internal security lens converged on the same flaw: the duplicate got the recent interaction BECAUSE of the bug, so recency legitimizes the wrong copy. **Converged:** evidence-ordered rules (pin → highest validated ownership epoch → server-registered live run), with backup-restore epoch re-seeding, forged-epoch rejection, and every genuinely ambiguous case escalating to a human rather than guessing.

**Originally**, the arbiters' guardrails were mostly prompt-level. **Converged:** the J2 survivor arbiter can propose an ownership move only with mechanical evidence that PREDATES the duplicate's spawn (a poisoned terminal tail or a self-minted post-spawn commitment can never win it ownership), deferrals are capped, all free text is enveloped as untrusted data, every call is provenance-logged under a net-new HTTP deny mechanism (the "existing denylist" the first draft cited turned out not to exist — reviewers caught the phantom and it is now a named build with symlink-proof tests), and the Judgment-Within-Floors standard itself now requires shadow-beats-static evidence before any arbiter acts.

**Originally**, the spec implied stronger consistency than the substrate provides. **Converged:** an explicit consistency contract (§3.0) states the safety and liveness guarantees AND the non-goals — no linearizable ownership, no guaranteed single owner under partition — each mapped to a bounded/loud/converge-back mechanism, with a partition/delayed-replay test matrix.
