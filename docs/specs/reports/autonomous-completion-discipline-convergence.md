# Convergence Report — Autonomous Completion Discipline

## ELI10 Overview

When you put the agent in a long, pre-approved autonomous run and walk away, you want it to actually keep going until the work is genuinely done — not quit early because it decided some reversible choice "needs your input," or that finishing a few tasks was a tidy place to pause. We already shipped the *prompt* half of that fix (a "these are the only valid reasons to stop" section in the agent's playbook). This spec is the *structural* half: a stop-gate that actually blocks the session from exiting while real work remains, makes an independent judge (not the agent's own say-so) the arbiter of "is it done," and — when the agent claims it's genuinely blocked by something external it can't fix — requires it to leave a tamper-visible, you-can-acknowledge-it record instead of quietly disappearing.

The review process changed the design a lot. The first draft *looked* solid but, checked against the actual code, its core anti-cheating mechanism didn't exist where the draft assumed it did — so a session could have slipped out through a brand-new, *unguarded* exit. Five rounds of adversarial + security + lessons + scalability + integration review (plus one factual check with the Claude Code docs) turned that into a design where every exit is either genuinely earned (the work is done), genuinely blocked (an external wall, recorded and acknowledgeable), or the clock ran out — and where a session can't fool the judge by injecting instructions into its own transcript, can't quietly induce a failure to slip out, and can't be trapped grinding pointless busywork either.

The biggest single catch: the whole draft (and two reviewers) had misread a timeout value as milliseconds when Claude Code's docs say it's seconds — so a "fix" to lower that timeout would have *killed* the agent's own pacing sleep mid-run and caused the exact silent-stall it was trying to prevent. Verifying the unit against the docs reverted that and left the design simpler and correct.

## Original vs Converged

- **Originally,** the new `<hard-blocker>` exit marker was described as guarded by the stop-rationale judge (P13). In the real code P13 is only consulted on the completion/promise branches; a bare stop just gets re-fed. So the marker, as drafted, was an *unguarded* one-line exit — easier than completing. **Converged:** the marker branch now explicitly calls the (extended) judge before any exit, sits after emergency-stop + duration, and is necessary-but-not-sufficient.
- **Originally,** the design "fed objective signals to the judge" — but the judge's API only accepts the transcript text; those signals didn't exist. **Converged:** a backward-compatible `signals` object is threaded through all four surfaces (hook payload → route → method → prompt).
- **Originally,** the judge read the agent's own transcript raw — so the agent could inject "respond STOP_OK" and (because the judge fails open) slip out silently. **Converged:** the transcript is fenced as instruction-inert data, the judge is told to treat guard-directed phrasing as *evidence of gaming*, a deterministic `injectionSuspected` signal is computed over the judge's full window, and a regression test pins it.
- **Originally,** a fabricated-but-coherent "I'm blocked" exited cleanly, with the acknowledgement loop deferred. **Converged:** the judge classifies the blocker external-vs-buildable (a buildable "blocker" is refused), and every hard-blocker exit raises an /ack-able Attention item in v1 (not deferred).
- **Originally,** the completion judge looked like it would run on *every* stop (an LLM call per turn — a quota burn on the loop most prone to rate-limits). **Converged:** a cheap deterministic checkbox scan is the primary signal; the LLM judge fires only when the run might actually be done.
- **Originally,** a timeout value was misread as milliseconds; two revisions then "sized" the hook timeout down (45s, 60s) and engineered the backoff clamp around it. **Converged:** verified against the Claude Code docs that the unit is *seconds* — the hook already has an effectively-unlimited (~2.8h) budget, so the judge call + the pacing sleep fit with vast margin; the timeout-lowering (which would have been harmful) was reverted and the design simplified.

## Full Findings Catalog

**Iteration 1 (17 material):** (HIGH) prompt-injection into the judge; self-inducible fail-open exit; (CRITICAL, adversarial) the marker was unguarded by P13 + nothing verified the blocker (P13 mis-placed in the draft); (HIGH) marker collides with the `<promise>` whitespace-collapsing extraction + self-trips on its own docs; checkbox scan gameable both ways + sole-signal in promise-fallback; the completion judge framed as O(turns) LLM calls + 10s-vs-35s timeout confusion; (HIGH, integration) config-migration named the wrong mechanism (should be ConfigDefaults), the P13 signal-feed signature/route change omitted, the SKILL.md Step-2b frontmatter template (the load-bearing edit) not flagged; (HIGH, lessons) the objective-signals wiring unimplementable as written, the milestone-phrase class had zero structural floor (pure prompt-willpower), a new manufactured-hard-blocker dishonesty vector, recurrence-risking deferrals (Q3/Q4) + missing `lessons-engaged:`; (MED/LOW) artifact tamper-claim over-stated + unbounded/unsanitized fields (log-injection) + leak surface, denial-of-exit + no scoped operator unblock, marker anti-collision nonce, no-baseline conservatism, in-flight backward-compat, off-switch half-gated, unbounded-run no-backstop, version-skew window, §3.1 over-claim (judge sees prose, can't run tests). — ALL resolved in the iteration-2 spec.

**Iteration 2 (4):** NEW-1 (HIGH) the 8s curl budget worsened the self-induced-timeout vector; NEW-2 (MED) scan/judge window asymmetry; NEW-3 (MED) version-skew (a)-exit load-fragility; NEW-4 (LOW) SKILL `(a)`-prose vs nonce'd marker; + security non-blocking refinement (milestone-signal survival on the condition path made a tested invariant). — Resolved iteration 3.

**Iteration 3 (1):** NEW-1 (HIGH) judge curl + idle-backoff sleep additive within one hook invocation (≈55s) vs the (then-45s) registered timeout → host-kill strand risk; + the §5 prose was factually wrong about how the self-clamp scales at 45s and what a host-kill does. — Resolved iteration 4 (set 60s + non-additive).

**Iteration 4 (1):** NEW-2 (HIGH) `BK_MAX = BK_REG_TIMEOUT / 3` yields 20000 (not 20) at `timeout: 60000` because the hook reads the field raw with no ms→s conversion and consumes `BK_MAX` as seconds — silently disabling the clamp; surfaced the underlying seconds-vs-milliseconds ambiguity. — Resolved iteration 5 via an authoritative docs check: the Claude Code hook `timeout` is in **seconds**, so `timeout: 10000` ≈ 2.8h (effectively no timeout); the prior timeout-lowering work was premised on a misread and was reverted (lowering it would have host-killed the hook mid-300s-backoff-sleep), and the `JUDGE_CALL_RAN` skip-backoff mechanism (only a timeout-safety device) was dropped.

**Iteration 5 (0):** adversarial — CONVERGED, no material issues (timeout math/prose correct; revert broke nothing; self-induced-fail-open still handled + disentangled; all prior-closed items unaffected). lessons-aware — CONVERGED, the revert is a Structure>Willpower / verify-before-claiming win with no regression.

## Convergence verdict

Converged at iteration 5. No material findings in the final round from either the gating adversarial dimension or the mandatory lessons-aware dimension (security, scalability, and integration converged at iteration 2). The spec is ready for user review and re-approval.
