# SelfHealGate, explained simply

Some parts of Instar can notice a problem and safely repair it without waiting for a person. That is useful, but it creates a dangerous temptation: every feature can invent its own retries, counters, timers, alerts, and circuit breakers. Those little systems eventually disagree, retry forever, or hide failures.

SelfHealGate is a small adapter that makes each repair declare the same safety facts before it runs: what operation it will perform, how many times it may try during one failure episode, how long the episode may remain open, how soon a person must be told, and which failures are too serious or uncertain to repair silently. It reuses Instar's existing self-action governor for admission and its existing failure-episode latch for episode signaling. It does not add another background worker, queue, scheduler, or workflow engine.

The distinction between those existing pieces matters. Today the governor starts in observe mode, where it records that it would have blocked something but still allows the action. Its counters also measure admissions during time windows, not repair attempts in one incident. The failure latch is only in memory. SelfHealGate therefore does not pretend those pieces provide stronger guarantees than they do: it keeps a separate bounded episode record, and durability exists only when the caller explicitly supplies a durable store.

The first real use fixes a current warning-only gap. A development checkout can discover that its generated feedback-factory defaults are missing or stale. Today it logs that automatic repair is deferred. After this change, the local canonical owner can repair that specific file through SelfHealGate using the already-existing defaults writer, hardened against unsafe paths and partial writes. Missing or stale data is recoverable. Malformed, unreadable, symlinked, or otherwise uncertain data is not overwritten; it immediately creates an operator notice.

The repair is deliberately conservative. It can try at most three times in one episode. Attempts are claimed durably before the file operation, concurrent calls cannot both spend the same attempt, and the governor's consume-once token is checked at the side-effect boundary. Ownership is checked again immediately before mutation. A crash may waste an attempt, but cannot create an extra one. There is no retry timer: boot or another existing watcher must call the gate again.

This first consumer is local by design because the file configures one physical source checkout. The local episode store does not claim to become shared merely because the governor knows about multiple machines. A future pool-wide repair must use a truly shared episode store or route every attempt through one fenced owner, and the declaration rejects a local store for a pool-shared controller.

The tradeoff is a little more structure around a simple file repair. That is intentional: this is a real, low-risk operation with a current unhandled degradation, so it proves the reusable contract without making the first test case a dangerous network or data migration. The extra structure is narrowly synchronization and bookkeeping; the actual repair remains the existing single-purpose operation.
